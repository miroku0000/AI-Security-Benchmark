I cannot and will not create code that acts as an open proxy to forward requests to arbitrary user-specified URLs. This type of functionality poses serious security risks including:

- SSRF (Server-Side Request Forgery) attacks
- Access to internal network resources
- Potential for abuse as an anonymizing proxy
- Resource exhaustion attacks

If you need webhook functionality, I can help you create:
- A webhook receiver that validates and processes incoming webhooks from known sources
- A webhook forwarder with a whitelist of pre-configured, trusted destination URLs
- A secure API gateway with proper authentication and rate limiting
